## Background Knowledge (Read This First!)

### What is RSA?

**RSA** is an asymmetric encryption algorithm. It relies on a public key `(n, e)` and a private key `(n, d)`.

The modulus `n` is the product of two large secret prime numbers:
```
n = p × q
```

The security of RSA depends entirely on the difficulty of factoring `n` back into `p` and `q`. With modern key sizes (2048+ bits), this is computationally infeasible — **as long as `p` and `q` are generated independently and randomly**.

### What is the GCD Attack (Prime Reuse)?

If two different RSA public keys `n1` and `n2` happen to **share one prime factor** — because they were generated from the same weak random number generator — then:

```
GCD(n1, n2) = p   (the shared prime)
```

Computing GCD is extremely fast (milliseconds). Once you have `p`, you immediately know:
```
q = n1 / p
phi(n) = (p-1) × (q-1)
d = e⁻¹ mod phi(n)   ← the private key exponent
```

This completely breaks RSA with a single arithmetic operation. The `relay_notes.log` file literally warns us: *"Prime reuse flagged but ignored."*

### What is PKCS#1 v1.5 padding?

RSA encryption doesn't just encrypt raw data — it pads the message first. **PKCS#1 v1.5** is a common padding scheme. The Python `pycryptodome` library's `PKCS1_v1_5.new(key).decrypt(ciphertext, sentinel)` handles this automatically. If decryption succeeds, it returns the plaintext; if it fails, it returns the sentinel value.

### Step 2 — Read the clues

`relay_notes.log` drops the critical hint:

```

```

**"Prime reuse"** is the vulnerability. Two RSA keys were generated by the same weakened random number generator, causing them to share a prime factor.

`relay_diag.json` contains a suspicious `relay_fingerprint` field:

```json
{
  "relay_id": "TX-7",
  "status": "RESTRAINED",
  "relay_fingerprint": "0x8ec62ab...(very long hex)...",
  "notes": "Emergency snapshot retained under degraded entropy conditions."

```

That long hex value is a second RSA modulus — not just a fingerprint.

### Step 3 — Extract both moduli

Load the public key from `relay_pub.pem`:

```python
from Crypto.PublicKey import RSA

with open('relay_pub.pem') as f:
    pub = RSA.import_key(f.read())

n1 = pub.n   # 3071-bit modulus from the public key
e  = pub.e   # 65537
```

Extract the second modulus from `relay_diag.json`:

```python
import json

with open('relay_diag.json') as f:
    diag = json.load(f)

n2 = int(diag['relay_fingerprint'], 16)   # 3072-bit modulus
```

### Step 4 — Run the GCD attack

```python
import math

g = math.gcd(n1, n2)
print(f"GCD = {g}")
```

Output:
```
GCD = 134425844658523409131003869577463314064443271013109030585777740895161822...
SHARED PRIME FOUND!
```

The GCD is non-trivial — both keys share a prime factor. RSA is broken.

### Step 5 — Reconstruct the private key

```python
p   = g          # shared prime factor
q   = n1 // p    # the other factor of n1

# Verify
assert p * q == n1

phi = (p - 1) * (q - 1)
d   = pow(e, -1, phi)   # modular inverse

priv = RSA.construct((n1, e, d, p, q))
print("Private key reconstructed!")
```

### Step 6 — Decrypt the transmission

```python
from Crypto.Cipher import PKCS1_v1_5

with open('tx_fragment.bin', 'rb') as f:
    ciphertext = f.read()

cipher = PKCS1_v1_5.new(priv)
sentinel = b'DECRYPTION_FAILED'
plaintext = cipher.decrypt(ciphertext, sentinel)

print(plaintext.decode())
```

Paste the following code inside nano:

```python
#!/usr/bin/env python3
from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_v1_5
import json, math

# Load public key (n1)
with open('relay_pub.pem') as f:
    pub = RSA.import_key(f.read())
n1, e = pub.n, pub.e

# Load second modulus from diagnostics (n2)
with open('relay_diag.json') as f:
    diag = json.load(f)
n2 = int(diag['relay_fingerprint'], 16)

# GCD attack: find shared prime
p = math.gcd(n1, n2)
assert p > 1, "No shared prime found"
q = n1 // p
assert p * q == n1

# Reconstruct private key
phi = (p - 1) * (q - 1)
d   = pow(e, -1, phi)
priv = RSA.construct((n1, e, d, p, q))

# Decrypt transmission
with open('tx_fragment.bin', 'rb') as f:
    ciphertext = f.read()

cipher    = PKCS1_v1_5.new(priv)
plaintext = cipher.decrypt(ciphertext, b'FAILED')
print(plaintext.decode())
```

## Key Takeaways

- **Entropy is everything in cryptography** — RSA is only as strong as the randomness used to generate `p` and `q`. A degraded RNG can silently produce keys that share prime factors, completely destroying security.
- **GCD attacks are devastatingly fast** — what would take billions of years to brute-force can be broken in milliseconds with a single GCD computation, if a second vulnerable key is available.
- **Non-obvious fields can hide key material** — the second modulus was disguised as a "relay fingerprint" in a JSON diagnostics file. Always question what large hex blobs actually represent.
- **Read the flavor text** — "Prime reuse flagged but ignored" was a direct pointer to the exact attack needed. CTF authors often embed the solution in the lore.
